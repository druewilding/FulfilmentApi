## What is an invariant?
- An invariant is a rule that must always be true about an object.
- Example invariants from the Order:
  - Total weight must not exceed 30kg
  - Items cannot be added to a confirmed order
  - Only a pending order can be confirmed
- Mathematics terminology: something that does not vary.
- In domain modelling, invariants make aggregate roots valuable. The object itself guarantees that the rules are never broken.
- Unlike Rails, that can bypass with `save(validate: false)` .NET aggregate roots _cannot_ be put into an invalid state.
