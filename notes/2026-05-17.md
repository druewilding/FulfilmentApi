## Clean code layer boundaries
- The domain needs to stay pure - no MassTransit references there.
- The layer boundaries are like this:
  - **API layer** - HTTP endpoints, middleware, wiring together
  - **Domain layer** - business rules, aggregates, value objects
  - **Infrastructure layer** - talks to external systems (databases, message brokers, email, etc)
- Add MassTransit to the Infrastructure project.

## Write a message consumer
- An `InventoryReservationConsumer` will listen for `OrderPlaced` event and react to it.
- A consumer is a class that implements `IConsumer<T>`.
- The `Consume()` function consumes messages.
- There is an automatic ack/nack based on the outcome of consuming.
- Write a comsumer in the Infrastructure project.
- Register it in the Program so that MassTransit knows it exists.
- The `using` command must match the `namespace` used in the consumer file.
- I saw a purple line in the RabbitMQ Management UI for "Consumer ack" 😊
- Add a `DeliverySchedulingConsumer` in the exact same pattern.
- Now both consumers log a message in the console.
- Fan-out pattern:
  - both consumers bind to the same exchange
  - each gets their own queue
  - each gets an independent copy of the message
  - one publish triggers both
