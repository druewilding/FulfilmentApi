## Domain modelling
- We are going to move things into `FulfilmentApi.Domain` - a separate project that contains pure business logic.
- An **aggregate root** is a domain object that enforces business rules and controls access to everything it owns. Root means it is the entry point. An Order is an entry point. OrderItem is not. You can tell an Order to add an item, and it decides whether it's allowed.
- You'll have a method like `order.Confirm()` that internally validates the rules and changes state. *The object protects its own invariants.*
- In C# terms:
  - Properties have private setters
  - State changes happen through methods
  - The object may throw an exception
- Primary constructor is great for dependency injection - not for an aggregate root. Use a regular constructor and control the get/set access.
- I made a class. made the get public, but the set private.
- Used an enum for the order Status.
- The public `Confirm()` method is what actually changes the order status. It enforces the rule. This is aggregate root in action.
